练习名称  
6to4隧道的使用
练习内容  

    各主机打开协议分析器，进入相应的网络结构并验证网络拓扑的正确性，如果通过拓扑验证，关闭协议分析器继续进行实验，如果没有通过拓扑验证，请检查网络连接。
    本练习将主机A、B、C、D、E、F作为一组进行实验。
1.  配置主机B为6to4路由器：
    （1）检验是否已启动IPv6协议栈，若没有启动，则安装IPv6协议栈；若已启动，则重置IPv6设置：
    ① netsh int ipv6 reset。
    ② 禁用主机B的第二块网卡<与主机E在同一网段的网卡>（控制面板/网络连接/网卡名称/单击右键/禁用)。
    ③ 启用主机B的第二块网卡<与主机E在同一网段的网卡>（控制面板/网络连接/网卡名称/单击右键/启用）。
    （2）启动6to4（netsh int ipv6 6to4 set sta sta=enabled und=disabled ）。
    （3）在物理接口1上启用转发和通告
        （netsh interface ipv6 set interface 物理接口1的索引值
         forwarding=enabled advertise=enabled）。
    （4）在物理接口2上启用转发
        （netsh interface ipv6 set interface 物理接口2的索引值 forwarding=enabled）。
    （5）在6to4 Pseudo-Interface上启用转发
        （netsh interface ipv6 set interface 3 forwarding=enabled）。
    （6）向物理接口1添加6to4前缀的路由，并进行公布
        （netsh interface ipv6 add route 2002:dd08:3dac:1::/64 物理接口1的索引值 publish=yes）。
2.  配置主机E为6to4路由器：
    （1）检验是否已启动IPv6协议栈，若没有启动，则安装IPv6协议栈；若已启动，则重置IPv6设置：
    ① netsh int ipv6 reset。
    ② 禁用主机E的第一块网卡<与主机B在同一网段的网卡>（控制面板/网络连接/网卡名称/单击右键/禁用）。
    ③ 启用主机E的第一块网卡<与主机B在同一网段的网卡>（控制面板/网络连接/网卡名称/单击右键/启用）。
    （2）启动6to4（netsh int ipv6 6to4 set sta sta=enabled und=disabled ）。
    （3）在物理接口1上启用转发
        （netsh interface ipv6 set interface 物理接口1的索引值 forwarding=enabled）。
    （4）在物理接口2上启用转发和通告
        （netsh interface ipv6 set interface 物理接口2的索引值
         forwarding=enabled advertise=enabled）。
    （5）在6to4 Pseudo-Interface上启用转发
        （netsh interface ipv6 set interface 3 forwarding=enabled）。
    （6）向物理接口2添加6to4前缀的路由，并进行公布
        （netsh interface ipv6 add route 2002:dd08:3dad:1::/64 物理接口2的索引值 publish=yes）。
3.  查看主机B、E的地址信息、路由信息：
    netsh int ipv6 show add（着重查看接口3）
    netsh int ipv6 show rou
    ●  找出与6to4隧道相关的地址信息和路由信息条目。
4.  配置主机A：
    ① 执行重置设置（netsh int ipv6 reset; netsh int ipv6 renew）。
    ② 查看路由信息（netsh int ipv6 show route）。
    ③ 若在显示的路由信息中没有前缀为2002::/16的路由条目，则添加该路由条目
      （netsh int ipv6 add route 2002::/16 物理接口索引值 主机B（物理接口1）的链路本地地址）。
5.  配置主机F：
    ① 执行重置设置（netsh int ipv6 reset; netsh int ipv6 renew）。
    ② 查看路由信息（netsh int ipv6 show route）。
    ③ 若在显示的路由信息中没有前缀为2002::/16的路由条目，则添加该路由条目
      （netsh int ipv6 add route 2002::/16 物理接口索引值 主机E（物理接口2）的链路本地地址）。
6.  主机A、B、E、F启动协议分析器并开始捕获数据。
    注：主机B捕获通过物理网卡2的数据，主机E捕获通过物理网卡1的数据。
7.  主机A ping 主机F的6to4地址2002:dd08:3dad:1:XXXX:XXXX:XXXX:XXXX
    （主机F的6to4地址从地址本中获得）。
8.  主机A、B、E、F停止数据捕获，并进行数据分析：
    ●  在主机A捕获的数据中，有“6to4隧道”会话吗？在ICMPv6信息报文中的“回送请求/应答”会话中选第一个数据包，它的源IP地址____________________，目的IP地址___________________，主机A和主机B之间的通信是IPv6方式还是IPv4方式？
    ●  在主机B、E捕获的数据中，有“6to4隧道”会话吗？若有，取该会话中的第一个数据包。在IPv4层中，源IP地址_____________，目的IP地址____________；在IPv6层中，源IP地址__________________，目的IP地址__________________；说明是怎样将IPv6数据包封装成IPv4数据包的。主机B和主机E之间的通信是IPv6方式还是IPv4方式？
    ●  在主机F捕获的数据中，有“6to4隧道”会话吗？在ICMPv6信息报文中的“回送请求/应答”会话中选第一个数据包，它的源IP地址____________________，目的IP地址___________________，主机E和主机F之间的通信是IPv6方式还是IPv4方式？
    ●  由上面的分析结果，绘制出在6to4隧道机制下主机A ping主机F的数据交互图，以及6to4主机和6to4路由器、6to4路由器和6to4路由器、6to4路由器和6to4主机间的通信类型。
9.  复位主机A、B、C、D、E、F的IPv6配置状态：
    在各主机上(顺序是：主机B、E、A、C、D、F)分别执行命令：netsh int ipv6 reset。
    主机B、主机E分别执行命令：netsh int ipv6 6to4 reset，重置IPv6 6to4 状态。
