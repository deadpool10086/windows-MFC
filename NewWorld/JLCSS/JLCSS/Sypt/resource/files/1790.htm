练习名称  
模拟RARP协议
练习内容  

    各主机打开协议分析器，进入相应的网络结构并验证网络拓扑的正确性，如果通过拓扑验证，关闭协议分析器继续进行实验，如果没有通过拓扑验证，请检查网络连接。
    本练习将主机A、B、C、D、E、F作为一组进行实验。
1.  所有主机编写RARP协议脚本。
    （1）组内所有主机打开协议编辑器，选中视图菜单，进入协议生成视图。
    （2）打开树状编辑视图中的树型结构，将第二个validproto节点的name属性由temp修改为：RARP。
    （3）将proto的name属性修改为：RARP，showname_cn属性修改为：逆地址解析协议，showname_en：Rarp protocol。
    （4）右键点击proto分支下的fields节点，选择向前插入，插入一个preproto节点。
    （5）选中插入的preproto节点，在右侧的工具箱中的属性页签中，将preproto的name属性设为：Ethernet。
    （6）右键点击preproto节点，选择子节点，插入一个protoref节点。
    （7）选中插入的protoref节点，在右侧的工具箱中的属性页签中，将protoref的name属性设为：RARP。
    （8）右键点击protoref节点，选择子节点，插入一个expr节点。
    （9）选中插入的expr节点，在右侧的工具箱中的属性页签中，将expr的oper属性设为：eq。
    （10）右键点击expr节点，选择子节点，插入一个fieldref节点。
    （11）选中插入的fieldref节点，在右侧的工具箱中的属性页签中， 将fieldref的name属性设为：type-length。
    （12）右键点击expr节点，选择子节点，插入一个number节点。
    （13）选中插入的number节点，在右侧的工具箱中的属性页签中，将number的value属性设为：32821，这个值的十六进制表示为：0x8035。
    （14）右键点击"fields"找到"nextproto"节点，选择向前插入，插入一个fixed节点。
    （15）选中插入的fixed节点，在右侧的工具箱中的属性页签中，设置fixed的属性，其中：
        name属性：                htype
        showfmt属性为：           dec
        showname_cn属性为：       硬件地址类型
        showname_en属性为：       Hardware address type
        size属性为：              short
    （16）右键点击nextproto节点，选择向前插入，插入一个fixed节点。
    （17）选中插入的fixed节点，在右侧的工具箱中的属性页签中，设置fixed的属性，其中：
        name属性：                ptype
        showname_cn属性为：       协议地址类型
        showname_en属性为：       Protocol address type
        size属性为：              short
    （18）右键点击nextproto节点，选择向前插入，插入一个fixed节点。
    （19）选中插入的fixed节点，在右侧的工具箱中的属性页签中，设置fixed的属性，其中：
        name属性：                hlen
        showfmt属性为：           dec
        showname_cn属性为：       硬件地址长度
        showname_en属性为：       Hardware address length
    （20）右键点击nextproto节点，选择向前插入，插入一个fixed节点。
    （21）选中插入的fixed节点，在右侧的工具箱中的属性页签中，设置fixed的属性，其中：
        name属性：                plen
        showfmt属性为：           dec
        showname_cn属性为：       协议地址长度
        showname_en属性为：       Protocol address length
    （22）右键点击nextproto节点，选择向前插入，插入一个fixed节点。
    （23）选中插入的fixed节点，在右侧的工具箱中的属性页签中，设置fixed的属性，其中：
        name属性：                op
        showfmt属性为：           dec
        showmap属性为：           RARPCode
        showname_cn属性为：       操作码
        showname_en属性为：       Operation code
        size属性为：              short
    （24）右键点击nextproto节点，选择向前插入，插入一个fixed节点。
    （25）选中插入的fixed节点，在右侧的工具箱中的属性页签中，设置fixed的属性，其中：
        name属性：                shaddr
        showgrp属性为：           3
        showname_cn属性为：       发送端硬件地址
        showname_en属性为：       Sender hardware address
        showsep属性为：           -
        vector属性为：            6
    （26）右键点击nextproto节点，选择向前插入，插入一个fixed节点。
    （27）选中插入的fixed节点，在右侧的工具箱中的属性页签中，设置fixed的属性，其中：
        name属性：                sipaddr
        showfmt属性为：           dec
        showgrp属性为：           1
        showname_cn属性为：       发送端IP地址
        showname_en属性为：       Sender IP address
        showsep属性为：           .
        size属性为：              word
    （28）右键点击nextproto节点，选择向前插入，插入一个fixed节点。
    （29）选中插入的fixed节点，在右侧的工具箱中的属性页签中，设置fixed的属性，其中：
        name属性：                rhaddr
        showgrp属性为：           3
        showname_cn属性为：       接收端硬件地址
        showname_en属性为：       Receiver hardware address
        showsep属性为：           -
        vector属性为：            6
    （30）右键点击nextproto节点，选择向前插入，插入一个fixed节点。
    （31）选中插入的fixed节点，在右侧的工具箱中的属性页签中，设置fixed的属性，其中：
        name属性：                ripaddr
        showfmt属性为：           dec
        showgrp属性为：           1
        showname_cn属性为：       接收端IP地址
        showname_en属性为：       Receiver IP address
        showsep属性为：           .
        size属性为：              word
    （32）右键点击proto节点，选择子节点，插入一个showmap节点。
    （33）选中插入的showmap节点，在右侧的工具箱中的属性页签中，将showmap的name属性设为：RARPCode。
    （34）右键点击showmap节点，选择子节点，插入一个case节点。
    （35）选中插入的case节点，在右侧的工具箱中的属性页签中，设置case的属性，其中：
        show_cn属性为：           请求
        show_en属性为：           Request
        value属性为：             3
    （36）右键点击showmap节点，选择子节点，插入一个case节点。
    （37）选中插入的case节点，在右侧的工具箱中的属性页签中，设置case的属性，其中：
        show_cn属性为：           应答
        show_en属性为：           Reply
        value属性为：             4
    （38）右键点击showmap节点，选择子节点，插入一个case节点。
    （39）选中插入的case节点，在右侧的工具箱中的属性页签中，设置case的属性，其中：
        oper属性为：              1
        show_cn属性为：           未知
        show_en属性为：           Unknown
    （40）右键点击proto节点，选择子节点，插入一个showsum节点。
    （41）右键点击showsum节点，选择子节点，插入一个prtcl节点。
    （42）选中插入的prtcl节点，在右侧的工具箱中的属性页签中，将prtcl的attrib属性设为：showname。
2.  所有主机对RARP协议脚本进行联合编译。
    （1）在工具栏中点击[打开]按钮，打开netpdl.xml文件。
    （2）在右侧的工具箱中点击文件属性页，选中[Document1.xml]和[netpdl.xml]的复选按钮，选中Documnet1.xml条目，如图：

图23-3  联合编译

    点击[编译]按钮。
    （3）保存刚刚编辑的Document1.xml文件。
    （4）保存编译后的脚本文件，例如，保存为rarp_protocol.xml。
3.  主机C与主机E编写RARP请求帧。
    （1）主机C与主机E点击视图菜单，切换到协议编辑视图。
    （2）点击设置菜单，选择加载协议脚本，在弹出的加载协议脚本对话框打开刚刚编译保存的脚本文件rarp_protocol.xml，有效组默认为test。
    （3）在工具栏上点击[新建帧]按钮，新建一个Ethernet帧。
    （4）填写该帧的以太网协议首部，其中：
        源MAC地址：本机的MAC地址
        目的MAC地址：FFFFFF-FFFFFF（广播地址）
        协议类型或数据长度：8035（RARP协议，十进制表示为32821）
    （5）填写RARP协议头信息，其中：
        硬件地址类型：1
        协议地址类型：0800
        硬件地址长度：6
        协议地址长度：4
        操作码：3
        发送端硬件地址：本机的MAC地址
4.  主机A编写RARP应答帧。
    （1）主机A点击视图菜单，切换到协议编辑视图。
    （2）点击设置菜单，选择加载协议脚本，在弹出的加载协议脚本对话框打开刚刚编译保存的脚本文件rarp_protocol.xml，有效组默认为test。
    （3）在工具栏上点击[新建帧]按钮，新建一个Ethernet帧，编写对主机C的RARP应答。
    （4）填写该帧的以太网协议首部，其中：
        源MAC地址：本机的MAC地址
        目的MAC地址：主机C的MAC地址
        协议类型或数据长度：8035（RARP协议，十进制表示为32821）
    （5）填写RARP协议头信息，其中：
        硬件地址类型：1
        协议地址类型：0800
        硬件地址长度：6
        协议地址长度：4
        操作码：4
        发送端硬件地址：本机的MAC地址
        发送端IP地址：本机的IP地址
        接收端硬件地址：主机C的MAC地址
        接收端IP地址：10.0.0.1
    （6）在工具栏上点击[新建帧]按钮，新建一个Ethernet帧，编写对主机E的RARP应答。
    （7）填写该帧的以太网协议首部，其中：
        源MAC地址：本机的MAC地址
        目的MAC地址：主机E的MAC地址
        协议类型或数据长度：8035（RARP协议，十进制表示为32821）
    （8）填写RARP协议头信息，其中：
        硬件地址类型：1
        协议地址类型：0800
        硬件地址长度：6
        协议地址长度：4
        操作码：4
        发送端硬件地址：本机的MAC地址
        发送端IP地址：本机的IP地址
        接收端硬件地址：主机E的MAC地址
        接收端IP地址：10.0.0.2
5.  主机A设置触发条件。
    （1）选择操作菜单，然后单击设置触发条件，打开管理策略对话框。
    （2）点击[新建]按钮，新建一个触发策略，进行设置，如下：
        策略名：rarp_reply_host_c
    （3）点击添加操作，添加一个与操作。
    （4）点击添加条件，选择设置条件值，进行设置，如下：
        名称：MAC_next_type_con
        偏移：12
        长度：2
        值：80 35（上层协议为RARP协议）
    （5）点击添加操作，添加一个与操作。
    （6）点击添加条件，选择设置条件值，进行设置，如下：
        名称：RARP_type_con
        偏移：20
        长度：2
        值：00 03 （RARP请求帧）
    （7）点击添加操作，添加一个与操作。
    （8）点击添加条件，选择设置条件值，进行设置，如下：
        名称：MAC_1_con
        偏移：22
        长度：4
        值：主机C的MAC地址的前4个字节
    （9）点击添加条件，选择设置条件值，进行设置，如下：
        名称：MAC_2_con
        偏移：26
        长度：2
        值：主机C的MAC地址的后2个字节
    （10）点击下一步继续。
    （11）选择以自定义帧为模板，选择帧模板是刚刚编辑的对主机C的RARP应答帧，这里为第1帧。点击下一步，直至完成。
    （12）点击[新建]按钮，新建一个触发策略，进行设置，如下：
        策略名：rarp_reply_host_e
    （13）点击添加操作，添加一个与操作。
    （14）点击添加条件，选择设置条件值，进行设置，如下：
        名称：MAC_next_type_con
        偏移：12
        长度：2
        值：80 35（上层协议为RARP协议）
    （15）点击添加操作，添加一个与操作。
    （16）点击添加条件，选择设置条件值，进行设置，如下：
        名称：RARP_type_con
        偏移：20
        长度：2
        值：00 03 （RARP请求帧）
    （17）点击添加操作，添加一个与操作。
    （18）点击添加条件，选择设置条件值，进行设置，如下：
        名称：MAC_1_con
        偏移：22
        长度：4
        值：主机E的MAC地址的前4个字节
    （19）点击添加条件，选择设置条件值，进行设置，如下：
        名称：MAC_2_con
        偏移：26
        长度：2
        值：主机E的MAC地址的后2个字节
    （20）点击下一步继续。
    （21）选择以自定义帧为模板，选择帧模板是刚刚编辑的对主机E的RARP应答帧，这里为第2帧。点击下一步，直至完成。
    （22）选中策略rarp_reply_host_c和rarp_reply_host_e点击确定。
    （23）选择操作菜单，然后单击启动触发，下发触发策略。
6.  主机B，D，F使用协议分析器加载协议脚本。
    （1）主机B，D，F关闭协议编辑器，启动协议分析器。
    （2）点击设置菜单，选择加载协议脚本，在弹出的加载协议脚本对话框打开编译保存的脚本文件rarp_protocol.xml，有效组默认为test。
    （3）在弹出的是否关闭所有子窗口信息框上，点击[是]按钮，加载协议脚本。
    （4）主机B，D，F新建一个捕获窗口，点击开始捕获。
7.  主机C和主机E发送RARP请求帧。
    主机C和主机E发送刚刚编辑完成的RARP请求帧。
8.  主机B，D，F捕获并分析自定义协议数据帧。
    （1）主机B，D，F停止捕获，切换到协议解析页签，查看捕获到的概要信息为“逆地址解析协议”的数据帧。
    （2）分配给主机C或者主机E的IP地址是什么？
9.  所有主机恢复默认协议脚本
    点击设置菜单，选择恢复默认协议脚本，弹出“默认协议脚本将覆盖当前地协议脚本，是否继续？”的对话框，选择“是”恢复协议脚本。
