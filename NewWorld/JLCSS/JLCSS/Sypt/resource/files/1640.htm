练习名称  
ISATAP隧道的使用
练习内容  

    本练习将主机A、B、C、D、E、F作为一组进行实验。
1.  配置主机B为ISATAP路由器。
    首先，去掉主机B先前的配置：
    netsh int ipv6 reset
    netsh int ipv6 6to4 set state state=disabled
    配置主机B为ISATAP路由器：
    （1）启动ISATAP：
        netsh int ipv6 isatap set sta enabled
        netsh int ipv6 isatap set router 172.16.1.1 enabled
    （2）在物理接口1上启用转发
        （netsh interface ipv6 set interface 物理接口1的索引值 forwarding=enabled）。
    （3）在物理接口2上启用转发
        （netsh interface ipv6 set interface 物理接口2的索引值 forwarding=enabled）。
    （4）在 Automatic Tunneling Pseudo-Interface上启用转发、公告功能 
        （netsh int ipv6 set int 2 forw=enabled adv=enabled）。
    （5）向Automatic Tunneling Pseudo-Interface添加ISATAP前缀的路由，并进行公布
        （netsh int ipv6 add rou 3ffe:ffff:0:1::/64 2 pub=yes）。
    （6）添加默认路由，下一跳地址为主机E的物理接口1的链路本地地址
        （netsh int ipv6 add rou ::/0 物理接口2的索引值
         nexthop=主机E的物理接口1的链路本地地址 pub=yes）。
2.  配置主机E为ISATAP路由器：
    首先，去掉主机E先前的配置：
    netsh int ipv6 reset
    netsh int ipv6 6to4 set state state=disabled
    配置主机E为ISATAP路由器：
    （1）启动ISATAP：
        netsh int ipv6 isatap set sta enabled
        netsh int ipv6 isatap set router 172.16.3.1 enabled
    （2）在物理接口1上启用转发
        （netsh interface ipv6 set interface 物理接口1的索引值 forwarding=enabled）。
    （3）在物理接口2上启用转发
        （netsh interface ipv6 set interface 物理接口2的索引值 forwarding=enabled）。
    （4）在Automatic Tunneling Pseudo-Interface上启用转发、公告功能
        （netsh int ipv6 set int 2 forw=enabled adv=enabled）。
    （5）向Automatic Tunneling Pseudo-Interface添加ISATAP前缀的路由，并进行公布
        （netsh int ipv6 add rou 3ffe:ffff:0:2::/64 2 pub=yes）。
    （6）添加默认路由，下一跳地址为主机B的物理接口2的链路本地地址
        （netsh int ipv6 add rou ::/0 物理接口1的索引值
         nexthop=主机B的物理接口2的链路本地地址pub=yes）。
3.  查看主机B、E的地址信息、路由信息：
    netsh int ipv6 show add（着重查看接口2）
    netsh int ipv6 show rou
    ●  找出与ISATAP隧道相关的地址信息和路由信息条目。
4.  配置ISATAP主机：
    （1）重起主机A、F的IPv6协议栈（netsh int ipv6 renew）。
    （2）主机A、F启动ISATAP（netsh int ipv6 isatap set sta enabled）。 
    （3）主机A、F启动协议分析器并开始捕获数据。
    （4）主机A、F指定ISATAP路由器
         （对于主机A：netsh int ipv6 isatap  set router 172.16.1.1 enabled;
          对于主机F：netsh int ipv6 isatap set router 172.16.3.1 enabled）。
    （5）主机A、F停止捕获数据：
    ●  进入“ISATAP隧道”会话选中第一个报文，该报文封装了哪种类型的IPv6数据包？IPv4层的源IP地址为______________，目的IP地址为______________；IPv6层的源IP地址为_____________________，目的IP地址为____________________。
    ●  进入“ISATAP隧道”会话选中第二个报文，该报文封装了哪种类型的IPv6数据包？IPv4层的源IP地址为______________，目的IP地址为______________；IPv6层的源IP地址为_____________________，目的IP地址为____________________；该报文公告了哪些信息？
5.  主机A、B、C、D、E、F在命令行下，分别输入命令：netsh int ipv6 show add。
    ●  观察“接口2：Automatic Tunneling Pseudo-Interface”地址项的变化。
    ●  经分析ISATAP地址可知，ISATAP地址使用本地管理的接口标识的通用形式为_____________________________。
6.  主机A、B、E、F启动协议分析器并开始捕获数据。
    注：主机B捕获所有物理网卡数据、主机E捕获所有物理网卡数据。
7.  主机A ping 主机F的ISATAP地址（3ffe:ffff:0:2::5efe:172.16.3.2）。
8.  主机A、B、E、F停止数据捕获，并进行数据分析：
    ●  在主机A捕获的数据中，有“ISATAP隧道”会话吗？若有，取该会话中的第一个数据包。在IPv4层中，源IP地址_____________，目的IP地址_________；在IPv6层中，源IP地址_________________，目的IP地址________________；主机A和主机B之间的通信是IPv6方式还是IPv4方式？
    ●  在主机B、E捕获的数据中，有“ISATAP隧道”会话吗？在“回送请求/应答”会话中选第一个数据包，它的源IP地址____________________，目的IP地址___________________，主机B和主机E之间的通信是IPv6方式还是IPv4方式？
    ●  在主机F捕获的数据中，有“ISATAP隧道”会话吗？若有，取该会话中的第一个数据包。在IPv4层中，源IP地址_____________，目的IP地址____________；在IPv6层中，源IP地址_________________，目的IP地址________________；主机E和主机F之间的通信是IPv6方式还是IPv4方式？
    ●  由上面的分析结果，绘制出在ISATAP隧道方式下主机A ping主机F的数据交互图，以及ISATAP主机和ISATAP路由器、ISATAP路由器和ISATAP路由器、ISATAP路由器和ISATAP主机间的通信类型。
9.  结合练习一和练习二的结果，找出两种隧道方式的不同点，说明各自适用的场合。
10. 复位主机A、B、C、D、E、F的IPv6配置状态：
    在各主机上(顺序是：主机B、E、A、C、D、F)分别执行命令：netsh int ipv6 reset。
    主机A、B、E、F分别执行命令：netsh int ipv6 isatap set state state=default来恢复IPv6 ISATAP到默认状态。
    主机B、主机E分别执行命令：netsh int ipv6 6to4 reset，恢复IPv6 6to4配置到默认状态。
