练习名称  
IPv6数据包的端到端传输过程
练习内容  
    各主机打开协议分析器，进入相应的网络结构并验证网络拓扑的正确性，如果通过拓扑验证，关闭协议分析器继续进行实验，如果没有通过拓扑验证，请检查网络连接。
    本练习将主机A、B、C、D、E、F作为一组进行实验。
1.  主机A、F清空目标高速缓存：
    在各自主机上，输入以下命令：
    netsh int ipv6 del dest
    记录当前目标高速缓存的表项：
    netsh int ipv6 show dest
2.  在主机BV、EV上停止静态路由、其它动态路由协议（routing stop）。
3.  主机BV、EV察看各自的路由表，并分析可到达的方向（ip -6 rou）。
4.  在主机BV、EV上启动静态路由（routing static）。
    ●  主机BV、EV察看各自的路由表，与步骤3的结果相比有何不同？
    ●  结合该步骤的操作和路由表的变化，说明静态路由的作用。
5.  主机A、F察看各自的路由表，并分析可到达的方向（netsh int ipv6 show rou）。
6.  在主机A上，输入以下带有-d选项的tracert命令，来跟踪主机A和主机F之间的路由：
    tracert -d 主机F的站点本地地址
    ●  分析主机A到主机F的路径。
7.  在主机F上，输入以下带有-d选项的tracert命令，来跟踪主机F和主机A之间的路由：
    tracert -d 主机A的站点本地地址
    ●  分析主机F到主机A的路径。
8.  主机A、F察看目标高速缓存：
    在各自主机上，输入以下命令：
    netsh int ipv6 show dest
    ●  与步骤1相比增加了哪些表项？
    ●  从表项中，可得到哪些信息？
9.  由以上操作及实验结果，请说明路由表、目标高速缓存的作用？描述源主机到目的主机（中间经由路由器）的数据传输过程。
